# TIL 26/02/27 - SQL

## SQL

> Structured Query Language
> 	- DBMS에서 사용하는 데이터 표준 질의 및 처리 언어
> 	- 독자적 문법을 갖는 DB 표준 언어

### SQL의 특징

 - 다른 언어보다 배우고 사용하기가 쉽다.
 - 집합적, 선언적 언어이다
 - ANSI, ISO 표준(DBMS 종류와 상관없이 사용 가능)
 - RDBMS와 함께 탄생, 발전
 - SEQUEL에서 유래

### SQL의 분류

- **DML(Data Manipulation Language)**
	- 데이터 조작어
	- `SELECT`, `INSERT`, `UPDATE`, `DELETE`
- **DDL(Data Definition Language)**
	- 데이터 정의어
	- `CREATE`, `DROP`, `ALTER`, `TRUNCATE`
- **DCL(Data Control Language)**
	- 데이터 제어어
	- `GRANT`, `REVOKE`
- **TCL(Transaction Control Language)**
	- 트랜잭션 처리어
	- `COMMIT`, `ROLLBACK`, `SAVEPOINT(Oracle에서만 사용)`

### SQL 문장의 구성요소

- 예약어, 연산자, 함수, 표현식, 조건
- 일반적인 작성 방법
	- 데이터 컬럼은 소문자로 작성
	- 데이터 타입, 제약조건은 대문자로 작성

**제약조건**
- NOT NULL
- DEFAULT
- PRIMARY_KEY()
- CONSTRAINT
- UNIQUE()
- CHECK

**타입**
> `VARCHAR`, `CHAR`
VARCHAR는 선언한 공간을 무조건 차지하지 않고 들어온 값 만큼만 차지한다.
CHAR는 선언한 공간을 무조건 사용한다.
> - VARCHAR는 데이터 크기에 따라 저장공간 크기가 변하는 가변길이 문자열 자료형이다.

### DDL(Data Definition Language)

> **CREATE**

`CREATE TABLE table_name;`

 - 데이터베이스 객체를 생성할 때 사용
 - 적절한 데이터 타입을 정해야 한다.

> **ALTER**

`ALTER TABLE table_name;`

- 테이블을 수정할 때 사용

ADD, MODIFY, CHANGE, DROP

> **TRUNCATE**

`TRUNCATE TABLE table_name;`

- 테이블에 있는 데이터 삭제
- DELETE와 다르게 복원 불가

> **DROP**

`DROP TABLE table_name;`

- 테이블 구조 자체를 제거한다


### DML(Data Manipulation Language)

> **SELECT**

`SELECT * FROM DEPT`

- 테이블, 뷰에 있는 데이터를 조회할 때 사용

> **INSERT**

- 테이블이나 뷰에 데이터 추가 시 사용

> **UPDATE**

- 테이블이나 뷰에 있는 데이터 수정할 때 사용

> **DELETE**

- 테이블이나 뷰에 있는 데이터 삭제할 때 사용


### DCL(Data Control Language)

> **GRANT**

`GRANT 권한명 TO 사용자 or ROLE명;`

- 권한 할당

> **REVOKE**

`REVOKE 권한명 FROM 사용자 or ROLE명;`

- 할당된 권한 해제

### TCL(Transaction Control Language)

데이터 일관성을 유지하면서 안정적으로 데이터를 복구하기 위해 사용

> **COMMIT**

- 변경된 내용을 영구 저장

> **ROLLBACK**

- 변경되기 이전 상태로 되돌림
