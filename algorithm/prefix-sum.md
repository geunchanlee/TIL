# 누적 합(Prefix Sum)

[백준 11659 구간 합 구하기 4](https://www.acmicpc.net/problem/11659) 문제를 보면 주어진 숫자 배열에서 특정 구간의 합을 여러 번 구해야 한다.

문제에서 주어진 예제 입력으로 살펴보자.

- 숫자 배열 `[5, 4, 3, 2, 1]`이 주어진다.
- 주어진 구간 `i`번째 수부터 `j`번째 수까지의 합을 구한다.
![[prefix-sum-1.png]]

- 구간 1과 3 사이의 합을 구한다(인덱스 0~2) 
- $5+4+3 = 12$
![[prefix-sum-2.png]]

- 구간 2와 4 사이의 합을 구한다(인덱스 1~3)
- $4+3+2 = 9$
![[prefix-sum-3.png]]


어떻게 구할까?

그림을 우리 눈으로 보면 쉽게 보인다. 코드를 짠다면 간단하게 생각해서 반복문으로 `i`와 `j` 사이를 더하는 코드를 만들면 될 것 같다. 

그런데 문제에 주어진 반복 횟수가 100,000번이다. 반복문을 사용해서 코드를 짠다면 같은 계산을 수 없이 반복하게 되어 시간이 오래 걸리게 된다.

해당 문제에서 주어지는 조건이 연산이 가장 많은 최악의 경우로 주어진다고 가정해보자.

`N`: 수 100,000개 주어짐
`M`: 합을 100,000번 구함
`i`, `j`: 1부터 100,000으로 가장 많은 계산을 수행하는 경우

**Python 코드**

⚠️ 매우 오래 걸림! 

```python
import random, time

# 랜덤 함수로 1~1000 숫자 100,000개 들어있는 리스트 만들기
arr = [random.randint(1,1000) for _ in range(100000)]

start = time.time()  # 시간 측정 
# 구간의 합 100,000번 구하기
for i in range(100000):
	section_sum = 0
	# 구간 합 1 ~ 100,000이 주어졌다는 가정
	for j in range(100000):
		section_sum += arr[j]
		
end = time.time()  # 시간 측정

print(f'실행시간: {end-start:.3f}초')

>>> 실행시간: 768.719초
```

실행시간을 측정하기 위해 코드를 실행하고 도저히 끝이 안 나서 한 숨 자고 돌아오니 768.719초 뒤에 실행이 완료되었다.

문제에서 주어진 1초의 시간 조건을 아득히 넘어간다.

이 문제를 해결해보자.

### 누적 합(Prefix sum)

> 숫자 배열에서 구간의 합을 효율적으로 구하려면 어떻게 할까?

위에서 반복문을 통해 구한 값들은 이미 한 번 구한 값을 계속 다시 구하느라 시간을 오래 사용한 것이다.
한 번 답을 구했다면 답을 답지에 저장해두고 필요할 때 가져다 사용한다면 쓸모없는 중복 계산을 줄일 수 있다.

처음의 예제로 돌아가서 배열의 합을 미리 저장해두고 구간의 합을 구해보자.

![[prefix-sum-4.png]]

예제에서 살펴본 구간 2와 4(인덱스 1~3)의 합 `9`를 아래의 정답 리스트를 활용해 구해보자. 

우선 주어진 리스트를 `arr`이라고 하고, 정답 리스트를 `ans`라고 하자.
`ans[3]`의 값 `14`는 다음과 같이 구해진다.
`14 = arr[0] + arr[1] + arr[2] + arr[3]`

그리고 구간 합 시작 범위 조건이었던 인덱스 1의 전 요소의 합인 `5`는
`5 = arr[0]` 이다.
`14 - 5 = 9`로 우리가 구하고자 한 arr 인덱스 1 ~ 3의 합과 같다.

![[prefix-sum-5.png]]

이미지로 이해하면 미리 구해둔 누적 합 `ans`리스트에서 우리가 구하고자 하는 구간의 끝인 인덱스3에 있는 값 `14`는 `arr[0] + (arr[1] + arr[2] + arr[3])`이다. 

즉, `파란색(14) = 초록색(5) + 빨간색(4+3+2)`이다.
이 내용을 이용해 정답 리스트를 통해 특정 구간의 합을 구할 수 있다.
문제에서 요구하는 구간의 합은 **빨간색 구간**을 구하는 것이다.

`빨간색 = 파란색 - 초록색`으로 구할 수 있게 된다. 좀 더 정리해보자.

---

우리가 구하고자 하는 구간을 `i`, `j`로 두고 `ans`에서 j를 찾으면 주어진 정수 리스트 `arr`의 `0 ~ j`번째 인덱스까지의 합을 알 수 있다.

여기서 인덱스 `i-1`을 `ans`에서 찾고, 방금 찾은 j번까지의 합에서 빼주면 인덱스 i부터 j까지의 합을 구할 수 있다.

일반화시키면 다음과 같다. 

$$arr[i] + arr[i+1] + \dots + arr[j]  =  ans[j] - ans[i-1]$$

이제 이 내용을 바탕으로 완료까지 시간이 10분도 넘게 걸리던 기존 코드를 개선해보자.

**Python 코드**

```python
import random, time

arr = [random.randint(1,1000) for _ in range(100000)]

start = time.time()

n = len(arr) + 1   # 인덱스를 1부터 사용하기
prefix = [0] * (n) # 정답 리스트 prefix 만들기

# prefix에 누적 합 저장
for i in range(1, n):
	prefix[i] = prefix[i-1] + arr[i-1]

# 구간의 합 찾기 1 ~ 100,000 구간의 합 100,000회
for i in range(100000):
	section_sum = prefix[100000] - prefix[0]

end = time.time()

print(f'실행시간: {end-start:.3f}초')

>>> 실행시간: 0.014초
```

코드의 실행시간이 기존 방법에서 소요되었던 768.719초에서 **0.014**초로 엄청나게 줄어들었다.
인덱스 1 ~ 100,000까지의 합이 `prefix[100000]`에 모두 들어있기 때문에 구간 합을 매우 빠르게 구할 수 있게 되었다.

시간복잡도로 생각해보면 기존 반복문 중첩 방식은 $O(N \times M)$의 시간복잡도를 갖는다.

개선된 누적 합 방식은 정답 리스트를 만들 때 $O(N)$, 구간 합을 조회할 때마다 $O(1)$이어서 총 $O(N + M)$으로 크게 줄어들었다.

이렇게 누적 합을 이용하면 매우 빠른 속도로 구간의 합을 구할 수 있다.

### prefix 누적 합 저장을 인덱스 1부터?

위의 코드 중간에 **"인덱스를 1부터 사용하기"** 가 있다. 왜 인덱스를 1부터 사용할까?

숫자 배열 `[5, 4, 3, 2, 1]`이 주어졌을 때, 구간 1~3(인덱스 0~2)의 합을 구한다고 하자.
이 경우 $5 + 4 + 3 = 12$ 가 답이다.
주어진 숫자 배열의 누적 합 배열 `prefix`는 `[5, 9, 12, 14, 15]`다.

위에서 구한 구간 합 공식은 $prefix[j] - prefix[i-1]$ 이다. 우리가 만약 `prefix`의 인덱스를 1부터 시작하지 않고 0부터 시작한다면 주어진 구간 합을 구하기 위해 $prefix[2] - prefix[-1]$ 을 해야 한다.

`prefix[2]`는 12, `prefix[-1]`은 15다. 공식을 적용한 결과가 $12 - 15 = -3$ 으로 정답이 아닌 값이 나온다. 시작 지점부터 구간 합을 구할 때의 문제를 해결하기 위해선 주어진 구간 `i`가 0인지 아닌지를 판별하는 코드가 추가적으로 필요하다. 

이렇게 불필요한 검사를 하지 않기 위해 `prefix`리스트의 맨 앞에 0을 추가해 누적 합을 1부터 저장시킨다.
구간 1 ~ 3을 다시 구해보자
- $i = 1, \ j=3$
- `prefix = [0, 5, 9, 12, 14, 15]`
- $prefix[3] - prefix[0]$ 을 사용해 답을 구한다.
- $12 - 0 = 12$

깔끔하게 답을 얻을 수 있다. 부가적으로 문제에서 주어진 1 ~ 3이라는 구간 조건과 사용하는 i, j가 일치하게 되어 직관적으로 알아보기도 더 쉬워진다.

> 인덱스를 1부터 시작해야 $prefix[j] - prefix[i-1]$ 공식을 사용해 모든 구간 합을 처리할 수 있다.
